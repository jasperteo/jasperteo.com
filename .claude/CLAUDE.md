# jasperteo.com

Personal website built with TanStack Start on Cloudflare Workers.

## Commands

```bash
bun run dev       # Dev server (port 5173)
bun run build     # Lint + production build
bun run preview   # Preview production build (port 4173)
bun run lint      # oxlint + ESLint
bun run fmt       # Format with oxfmt
```

## Tech Stack

- **Framework:** TanStack Start (React Router v3) + React 19
- **Runtime:** Cloudflare Workers (via `@cloudflare/vite-plugin`)
- **Styling:** Tailwind CSS v4 — no config file, CSS-first
- **UI Primitives:** `@base-ui/react` (headless, unstyled)
- **Component Variants:** `class-variance-authority` (CVA)
- **Class Merging:** `cn()` from `@/utils/utils` — uses `clsx` + `tailwind-merge`
- **Animation:** `motion`, `rough-notation`
- **Language:** TypeScript strict mode, ESNext

## Code Conventions

### TypeScript

- Strict mode — no `any`, no non-null assertions (`!`), no `as` casts unless unavoidable
- Use `type` for type definitions (not `interface`) — enforced by lint
- Use `import type` for type-only imports — enforced by lint
- No enums

### React

- Functional components only
- React Compiler is enabled (Babel plugin) — do not manually add `useMemo`/`useCallback` unless there's a specific reason
- `react-hooks/rules-of-hooks` is error, `exhaustive-deps` is warning — respect both

### Styling

- Tailwind CSS v4 — use CSS variables and `@theme` for design tokens, not `tailwind.config`
- Class order sorted by `oxfmt` (Tailwind sort enabled for `cn` and `cva` calls)
- Use `cn()` for conditional/merged class names
- Use `cva()` for component variants with multiple options

### Typography

Three fonts are available — use them intentionally:

| Utility      | Font       | Use for                                        |
| ------------ | ---------- | ---------------------------------------------- |
| `font-sans`  | Manrope    | Default — headings, labels, UI chrome, buttons |
| `font-geist` | Geist      | Body text, paragraphs, prose content           |
| `font-mono`  | Geist Mono | Code, monospace values, technical strings      |

- `font-sans` (Manrope) is the base font and applies by default — only set it explicitly when overriding another font
- `font-geist` (Geist) should be set on any container holding paragraph text or longer prose
- `font-mono` (Geist Mono) for inline code, code blocks, or any fixed-width output

### Accessibility

- **Use `@base-ui/react` primitives** for all interactive components — they handle ARIA roles, keyboard navigation, and focus management automatically
- **Semantic rendering** — when a button navigates, render it as the correct element using the Base UI `render` prop with `nativeButton={false}`: use `render={<Link to="..." />}` (TanStack Router) for internal navigation, or `render={<a href={...} target="_blank" rel="noopener noreferrer" />}` for external links
- **Decorative icons** — always add `aria-hidden="true"` to icon components that have adjacent visible text; this prevents the SVG `<title>` from being announced alongside the label
- **Icon-only interactive elements** — icon components include a `<title>` which screen readers can read, but prefer adding `aria-label` on the parent button/link for explicit, reliable labelling
- **`aria-label`** — use when the visible text alone isn't descriptive enough or when there is no visible text; do not duplicate visible text verbatim
- **Screen reader text** — use `<span className="sr-only">` for supplementary context not visible on screen (e.g. `", opens in new tab"` on external links)

### Components

- UI primitives live in `src/components/ui/` — built on `@base-ui/react` + CVA
- Icon components live in `src/components/icons/` — typed with `IconProps` from utils
- Use `data-slot` attributes on primitive wrappers (e.g. `data-slot="button"`) for parent selectors

### Routing

- File-based routing via TanStack Router — `src/routes/`
- `routeTree.gen.ts` is auto-generated — never edit manually
- Route param names and property order enforced by lint

### Imports

- Path alias: `@/` maps to `src/`
- Import order sorted and grouped by `oxfmt` / `eslint-plugin-simple-import-sort`

## Linting & Formatting

- **oxlint** — primary linter (TypeScript, React, Unicorn, TanStack Router rules)
- **ESLint** — runs after oxlint for rules not yet in oxlint
- **oxfmt** — formatter (tabs, trailing commas ES5, auto-sorts Tailwind classes and imports)
- `routeTree.gen.ts` is excluded from both lint and format

## Do Not

- Edit `src/routeTree.gen.ts` — it is auto-generated by TanStack Router
- Use `tailwind.config.ts` — this project uses Tailwind v4 CSS-first configuration
- Modify `wrangler.jsonc` without confirming — it controls production deployment config
